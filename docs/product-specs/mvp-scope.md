# WiseEff MVP 范围与版本切分

日期：2026-05-25

## 1. 切分原则

WiseEff 的正式开发应避免“一次性全栈大爆炸”。当前原型已经覆盖多个业务域，正式工程需要按可上线闭环拆分。每个里程碑都必须满足：可演示、可测试、可回滚、对用户有独立价值。

优先级：

1. 参数管理真实闭环。
2. 登录、权限、审计和后台基础设施。
3. 日志分析真实任务。
4. 设备调试网关。
5. 真实 Agent 编排。
6. 企业级上线治理。

## 2. M0：产品化基础

目标：让项目具备正式开发的工程地基。

范围：

- 建立后端服务骨架。
- 建立 PostgreSQL 数据库和迁移机制。
- 建立认证、用户、角色和权限表。
- 建立审计事件表和审计写入接口。
- 建立前端 API client 和 mock/production runtime 开关。
- 建立 CI：前端测试、后端测试、构建、类型检查。
- 建立本地开发环境：前端、后端、数据库、队列、对象存储模拟。

不包含：

- 真实日志 AI 分析。
- 真实设备写入。
- 真实 Agent 模型接入。

完成标志：

- 用户可以登录并看到基于后端返回的当前用户与权限。
- 前端仍可使用 mock，但生产配置不依赖 mock。
- 审计接口可以记录和查询事件。

## 3. M1：参数管理 MVP

目标：让参数管理成为第一个真实可用业务闭环。

范围：

- 项目、模块、参数定义、项目参数值、参数历史。
- 参数列表查询、搜索、筛选、分页和排序。
- 参数修改草稿与提交轮次。
- 参数变更审批状态机。
- 参数审阅、打回、推进、合入。
- 参数历史和审计。
- 批量导入的最小能力：上传、预览、校验、应用。
- 前端从 `mockData.ts` 迁移到 `ParameterRepository` 真实实现。

完成标志：

- 参数工作台、参数审阅、参数管理后台可以使用真实 API。
- 原型中的参数相关核心测试被保留或迁移为 API/E2E 覆盖。
- 参数变更合入后刷新页面仍保留真实状态。

## 4. M2：日志分析 MVP

目标：支持真实日志上传、后台分析任务和结果展示。

范围：

- 文件上传到对象存储。
- 日志记录、分析任务、阶段、证据、建议动作。
- 任务队列和 worker。
- 文本日志解析器。
- 基础规则分析器和可插拔 AI 分析接口。
- 进度查询或服务端事件。
- 前端从 mock 日志迁移到 `LogAnalysisRepository` 真实实现。

完成标志：

- 用户上传日志后可以看到真实进度。
- 失败格式返回可解释错误。
- 完成结果包含证据行号和建议动作。

## 5. M3：设备调试 MVP

目标：让调试平台通过安全网关完成设备节点读取、写入和回滚准备。

范围：

- 设备网关服务，隔离主业务后端和设备访问。
- 设备目标检测。
- 节点读取、写入、回读校验。
- 调试会话、快照和操作历史。
- 风险级别、访问模式、权限校验。
- 设备模拟器用于 CI 和本地测试。
- 前端从 mock 或本地 HDC wrapper 迁移到 `DebuggingGateway` 真实实现。

完成标志：

- 模拟设备环境下 E2E 通过。
- 高风险写入必须确认。
- 每次写入都有快照、回读结果和审计事件。

## 6. M4：Agent 协同 MVP

目标：让 Agent 从前端规则模拟升级为后端编排和受控工具调用。

范围：

- Agent 会话、消息、工具调用、审批记录。
- 页面上下文组装。
- 读工具：参数查询、日志解释、审计汇总。
- 写工具申请：参数草稿、审阅推进、日志重跑、设备写入准备。
- 人工审批和工具执行分离。
- Agent 输出持久化和审计。

完成标志：

- Agent 能基于真实数据总结和建议。
- Agent 发起写操作时只能生成待审批工具调用。
- 审批后执行结果可追溯。

## 7. M5：企业上线增强

目标：让 WiseEff 满足企业生产环境长期运行要求。

范围：

- SSO/OIDC 或企业身份源集成。
- 租户和组织隔离。
- 完整备份、恢复、数据保留和归档策略。
- 权限审批配置。
- 监控告警、容量指标、慢查询追踪。
- 安全扫描、依赖审计、运行时配置审计。
- 操作手册和事故响应流程。

完成标志：

- 可在 staging 和 production 环境独立部署。
- 有回滚方案和生产健康检查。
- 关键审计与安全要求通过验收。

Implementation note: the merged M5 scope is a controlled commercial pilot readiness baseline, not full enterprise production rollout. It implements pilot gates and production seams, while real staging smoke, HDC device-lab, backup/restore, rollback, live provider evidence, SSO/OIDC, durable queue, cloud SDK/IaC, monitoring, and capacity testing remain follow-up planning items.
