# DTS 拓扑鼠标/触控展开折叠设计

> English: [English](../../../superpowers/specs/2026-07-20-dts-topology-expand-collapse-design.md)

日期：2026-07-20  
状态：已确认

## 背景

`DtsTopologyNavigator` 已经保存逐节点展开状态、提供 `aria-expanded`、
隐藏被折叠的后代，并通过左右方向键实现 WAI-ARIA 树键盘模式。当前缺口在
鼠标和触控交互：箭头只是装饰，点击整行只会选择节点并筛选参数列表。

## 决策

有子节点的节点获得独立展开按钮。点击或触摸该按钮只展开或折叠子节点；
点击节点其余区域继续只负责选择节点和筛选参数。叶子节点不渲染展开按钮。

树节点仍是唯一的 roving-focus 焦点。展开按钮不会增加第二个 Tab 停靠点；
键盘用户继续在聚焦的树节点上使用左右方向键。鼠标/触控切换后，焦点返回
树节点，保证后续方向键导航稳定。

## 行为

- 展开状态：按钮读作“折叠 <节点>”，并渲染全部直接/间接可见后代。
- 折叠状态：按钮读作“展开 <节点>”，后代不出现在渲染树中。
- 点击展开按钮不会调用 `onSelectNode`，不会改变参数列表筛选。
- 点击树节点主体不会切换展开状态。
- 折叠选中节点的祖先不会清除选中身份；重新展开后恢复显示。
- 左右、上下、Home/End、Enter 和 Space 键行为保持不变。
- 触控布局下展开按钮至少为 44×44 CSS 像素，并提供明确的悬停/焦点样式。

## 组件设计

原生节点按钮调整为可聚焦的 `role="treeitem"` 容器，以便承载独立原生展开
按钮，同时避免 button 嵌套。展开按钮阻止点击事件冒泡，并复用现有
`setExpanded` 状态切换。节点容器保留 roving `tabIndex`、选中状态、
`aria-level`、`aria-expanded`、键盘处理和节点选择处理。

本次不修改 API、DTS 身份、树构建、数据库或提交工作流。

## 验证

组件测试必须证明：

1. 点击父节点展开按钮可以隐藏并恢复后代。
2. 点击展开按钮不会选择父节点。
3. 点击节点主体仍只选择，不切换展开状态。
4. 叶子节点没有展开按钮。
5. 既有键盘与 roving-focus 测试继续通过。

浏览器验收覆盖 `/parameters` 的 1440×900、768×1024、390×844，检查鼠标
展开折叠、参数筛选、控制台错误和页面级横向溢出。

## 文档影响

本设计补全 DTS 参数工作台既有设计中的逐节点展开入口，不改变 API 或领域
契约，因此无需更新 API、数据库 schema 或运行手册。
