# DTS 参数工作台方向纠偏与模块化重构 — 设计摘要（中文）

> 英文完整版：[English](../../../superpowers/specs/2026-07-20-dts-workbench-module-refocus-design.md)

## 问题

本轮工作台过度绑定 DTS 节点树与驱动展示，弱化了「按业务模块查看 / 修改 / 提交」的心智，并丢失批量提交、重要性筛选、历史 diff、跨项目对比、导出、列排序等易用能力。

## 决策

- 模块归属由**管理后台**维护（驱动 / compatible / 器件实例 → 模块）。
- **分期**：阶段一展示层重组 + additive 注册表（不改身份表）；阶段二下沉语义模型。
- 同名参数：模块 **或** 驱动不同即视为不同参数。
- API 模式红线不变：无 `recommendedValue`、无扁平 Excel。

## 阶段一要点

1. `/api/v2/parameter-modules` 注册表 + 映射。
2. 工作台默认「模块 → 器件/驱动 → 参数」导航；技术视图可选。
3. 主表：参数名 / 模块 / 器件·驱动 / 当前值 / 治理 / 操作。
4. 恢复重要性筛选、列排序、草稿多选、语义 CSV 导出。
5. 详情保留 provenance，并提供历史 / 跨项目对比占位。
6. 管理后台增加模块映射管理与未映射待处理队列。
